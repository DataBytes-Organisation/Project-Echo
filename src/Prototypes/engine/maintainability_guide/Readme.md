# Engine Prototypes Maintainability Guide

This directory stores the maintainability and handover materials created from the Sprint 1 prototype audit and extended in Sprint 2.

## Purpose
The goal of this guide is to help Engine team members and future contributors understand the `src/Prototypes/engine` directory more quickly and work from the correct current path.

## Current focus
The current semester implementation work is centred on `torch_impl`, which is treated as the main working area in the workbook.

## Included file
- `Project_Echo_Engine_Prototypes_Maintainability_Guide.xlsx`  
  A handover workbook that:
  - identifies current, reference, legacy, and deprecated-candidate areas
  - provides task-based navigation
  - separates `torch_impl` into dataset, model/config, validation, benchmarking, and deployment views
  - preserves detailed audit notes for justification and handover
